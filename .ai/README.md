# 🤖 .ai/ Directory

**이 디렉토리는 AI가 프로젝트를 빠르게 이해하고 작업을 이어갈 수 있도록 만든 컨텍스트 문서 모음입니다.**

---

## 📋 문서 목록

### 1. **PROJECT_CONTEXT.md** ⭐ (필독)
**프로젝트 전체 개요 및 컨텍스트**

- 프로젝트 목적, 역할, 현재 상태
- 시스템 아키텍처 및 기술 스택
- 파일 구조 및 각 파일의 역할
- 핵심 API 엔드포인트 명세
- 환경 변수 및 보안 정책
- GCP 배포 정보
- 로컬 개발 가이드
- 자주 발생하는 문제 및 해결 방법

**언제 읽어야 하나요?**
- 새로운 AI가 처음 프로젝트에 투입될 때
- 프로젝트 구조나 설계를 이해해야 할 때
- 배포 정보가 필요할 때

---

### 2. **WORK_LOG.md**
**작업 일지 및 변경 이력**

- 날짜별 완료된 작업 목록
- 기술적 결정 사항 및 이유
- 발생한 이슈 및 해결 방법
- Git 커밋 히스토리
- 프로젝트 통계
- 다음 작업 (TODO)
- 팀 커뮤니케이션 내역

**언제 읽어야 하나요?**
- 과거에 어떤 작업이 있었는지 확인할 때
- 특정 이슈의 해결 방법을 찾을 때
- 프로젝트 진행 상황을 파악할 때

---

## ✍️ 문서 업데이트 가이드

### 언제 업데이트해야 하나요?

1. **주요 기능 추가/변경**
   - `PROJECT_CONTEXT.md`의 해당 섹션 업데이트
   - `WORK_LOG.md`에 날짜별 작업 내역 추가

2. **버그 수정**
   - `WORK_LOG.md`의 "발생한 이슈 및 해결" 섹션에 추가

3. **배포 변경**
   - `PROJECT_CONTEXT.md`의 "프로덕션 배포 정보" 업데이트
   - `WORK_LOG.md`에 배포 일시 및 변경사항 기록

4. **기술 스택 변경**
   - `PROJECT_CONTEXT.md`의 "기술 스택" 테이블 업데이트
   - `WORK_LOG.md`에 변경 이유 및 마이그레이션 과정 기록

5. **API 명세 변경**
   - `PROJECT_CONTEXT.md`의 "핵심 엔드포인트" 섹션 업데이트
   - 외부 팀에 영향이 있다면 커뮤니케이션 내역도 기록

### 업데이트 원칙

- ✅ **명확하게**: 미래의 AI/개발자가 이해할 수 있도록
- ✅ **간결하게**: 핵심 정보만, 불필요한 내용 제거
- ✅ **구조적으로**: 마크다운 헤딩, 코드 블록, 표 활용
- ✅ **시간순으로**: 날짜와 함께 변경사항 기록
- ✅ **이유와 함께**: "무엇"뿐만 아니라 "왜" 기록

## 🔄 AI 작업 시작 시 체크리스트

새로운 AI가 이 프로젝트를 이어받을 때:

1. [ ] `PROJECT_CONTEXT.md` 전체 읽기
2. [ ] `WORK_LOG.md` 최근 작업 내역 확인
3. [ ] Git 로그 확인 (`git log --oneline -20`)
4. [ ] 현재 배포 상태 확인 (`curl http://34.71.58.225:8000/health`)
5. [ ] 로컬 환경 설정 (가상환경, 의존성 설치)
6. [ ] 간단한 테스트 실행 (`curl localhost:8000/health`)
7. [ ] 사용자의 작업 요청 이해 및 계획 수립
